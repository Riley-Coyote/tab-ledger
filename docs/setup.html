Setup Guide

Tab Ledger

Nothing lost. Everything findable.

Local-first analytics and knowledge base for Claude Code sessions. Indexes everything, makes it searchable, and gives your AI tools persistent memory.

01

What it does

Tab Ledger continuously indexes your Claude Code sessions and makes them queryable — by you, by your AI tools, and by autonomous agents.

  • Full-text search across every conversation you've ever had
  • Token usage, cost tracking, and tool analytics per session
  • AI-generated summaries with next steps, blockers, and decisions
  • Cross-session connection detection (temporal, branch, slug-based)
  • Project taxonomy mapping across all your repos
  • Works with Claude Code, Codex, OpenClaw, or any agent with shell access
02

Install

pip install tab-ledger
Requires Python 3.10+
03

First run

Three commands to go from zero to searchable.

tab-ledger index
Scans ~/.claude/projects/ and indexes all session JSONL files into ledger.db. Extracts tokens, costs, tools, models, durations.
tab-ledger build
Builds the knowledge base — maps sessions to projects, indexes messages, creates FTS5 search, detects cross-session connections.
tab-ledger search "your query here"
Search across everything. You're done.

Data is stored at ~/.tab-ledger/ — two SQLite databases, entirely on your machine.

04

Claude Code integration

Add Tab Ledger as an MCP server to give Claude Code native access to your session history.

~/.claude/settings.json
{ "mcpServers": { "tab-ledger": { "command": "tab-ledger", "args": ["mcp"] } } }

This exposes 8 tools directly to Claude Code:

kb_search kb_semantic kb_context kb_memory kb_session kb_projects kb_timeline kb_stats
05

CLI reference

Every command outputs JSON, making it pipeable into other tools and agents.

tab-ledger indexIndex Claude Code sessions from ~/.claude/projects/
tab-ledger buildBuild the full knowledge base (8 stages)
tab-ledger search "query"Full-text search (FTS5 syntax supported)
tab-ledger semantic "query"Semantic search via embeddings
tab-ledger statsToken, cost, and tool usage analytics
tab-ledger timeline <project>Chronological session history for a project
tab-ledger context <project>Continuation context (last session, next steps, blockers)
tab-ledger memory <project>Full continuity packet with semantic anchors
tab-ledger projectsList all indexed projects with stats
tab-ledger session <uuid>Detailed view of a single session
tab-ledger serveStart the web dashboard on localhost:7777
tab-ledger mcpRun stdio MCP server for Claude Code
tab-ledger refreshNightly KB refresh (skip AI summarization)

Add --project <name> to filter most query commands by project. Add --limit N to cap results.

06

Python API

For programmatic access from scripts, agents, or notebooks.

from tab_ledger import KnowledgeBase kb = KnowledgeBase() # Full-text search results = kb.search("websocket auth", project="vessel") # Continuation context for session handoff context = kb.get_continuation_context("polyphonic") # Full continuity packet memory = kb.get_memory_packet("sanctuary") # Statistics stats = kb.get_stats() kb.close()
07

How it works

Tab Ledger reads the JSONL session files that Claude Code already writes to disk. It parses them, extracts structured data, and builds a searchable knowledge base. Nothing leaves your machine.

Source
~/.claude/projects/*.jsonl
Storage
~/.tab-ledger/
Databases
ledger.db + knowledge_base.db
Search
SQLite FTS5 + optional embeddings

The build pipeline runs 8 stages: schema creation, project taxonomy mapping, message indexing, FTS5 build, AI summarization, cross-session linking, auxiliary data indexing, and verification.
